How to implement

This example is based on Issue 813. It relies on the treeNodeTemplate to customize the display of each node. The example uses simple buttons to handle basic functions for creating, copying and deleting nodes.

The example does not show a edit function because of the endless possibilities on how to implement this. One option could be to show an input field instead of the span with the name of the node. This could be triggered via button, double click or how your app enters edit mode.
